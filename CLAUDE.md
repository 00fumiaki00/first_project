## 今日学んだ概念（Day 1）

### プログラムの基本
- プログラム＝コンピュータへの命令書
- 3つの基本構造：順番に実行・条件で分岐・繰り返す
- コンピュータは0と1しか理解できない
- PythonはAIの道具が一番揃っている言語

### ツールの全体像
```
VS Code（作業場）
└── ターミナル（内蔵）
└── Claude Code（内蔵・あなた）

Python（言語）
└── ライブラリ（既製品の部品）
└── フレームワーク（土台ごと借りる）

API（外から借りるインフラ）
データベース（アプリの記憶）
GitHub（コードの倉庫・履歴書）
```

### 重要概念
- **変数**＝データに貼るラベル（箱ではなく名札）
- **関数**＝繰り返す作業を一回定義して呼び出す仕組み
- **クラス**＝材料と動作がセットになった設計図
- **インスタンス**＝設計図から作った実物
- **オブジェクト指向**＝現実世界の構造をそのままプログラムに持ち込む思想
- **データ構造**＝リスト・辞書・タプル・セットの4種類
- **非同期処理**＝待ち時間を無駄にしない仕組み（async/await）
- **API**＝誰かが作った関数を外部から呼び出す仕組み
- **ライブラリ**＝既製品の部品（pip installで入れる）
- **フレームワーク**＝土台ごと借りる仕組み
- **データベース**＝アプリの記憶（SQL/NoSQL）
- **クラウド**＝24時間動かし続ける場所
- **GitHub**＝コードの倉庫・デプロイの起点
- **デプロイ**＝倉庫から取り出して実際に動かすこと
- **認証**＝お前は誰かを確認する仕組み
- **セキュリティ**＝APIキーを隠す・パスワードを暗号化する

### 設計の思想
- 設計＝作るものを小さな部品に分解すること
- 全てのプログラムは「入力と出力」で説明できる
- シンプルに・シンプルに・説明できる状態にする

### Gitの概念
```
git status　→　変更を確認する
git add　　→　ステージに上げる
git commit　→　写真を撮る（変更を記録する）
git push　　→　GitHubに送る
```
- ブランチ＝試作する場所・本番を壊さない
- コミットメッセージは「何をなぜ変えたか」を書く

### コードの作法
1. 名前は意味のある言葉にする
2. コメントは「なぜ」を書く
3. 一つの関数は一つのことだけやる
4. 同じコードを繰り返さない
5. インデントを揃える

**まず動くコードを書く→動いたら整える**

## 最初に作るポートフォリオ

**業務メール自動生成AIツール**

現場を知っている人間が作ったAIツールは圧倒的に強い。

### 設計図
```
【入力】
受信したメールの内容
+ 今日の作業情報（配達先・集荷・シャーシ等）

【処理】
① データベースからパターンを呼び出す
② Claude APIに送る
③ AIがメールを生成する

【出力】
そのまま送れるメール文章
```

### バージョン設計
```
Version 1（最初・1ヶ月）→　情報を入力するとメールを生成する
Version 2（慣れたら）　→　パターンを登録・管理できる
Version 3（実務レベル）→　受信メールを読んで自動で文脈を判断する
```

### 他業種への展開
- 設定画面を作る（業種・会社情報を登録できる）
- 設定を変えるだけで別業種に対応できる
- トラック版→建設業版→医療版と展開できる
